# API Contract

The API is versioned under `/api/v1`. It is the stable contract for agents and
integrations; UI-shaped legacy endpoints may remain but should not be treated as
the external interface.

## Principles

- Public read APIs expose only published public content.
- Authenticated APIs accept either a Supabase browser session or a Bearer token.
- All OpenAI, Supabase service-role, and Resend secrets stay server-only.
- DTOs are citation-friendly and stable; do not leak private auth/profile fields.
- Public reads support `?lang=en|zh`, `lc_locale`, and `Accept-Language`.
  Responses include `locale`, `availableLocales`, and locale alternate URLs
  where applicable. Field names and stable identifiers remain English.

## Public Endpoints

- `GET /api/v1/projects`
- `GET /api/v1/projects/{handle}/{slug}`
- `GET /api/v1/projects/{handle}/{slug}/stops/{stop}`
- `GET /api/v1/projects/{handle}/{slug}/markdown`
- `GET /api/v1/projects/{handle}/{slug}/ai-visibility`

The markdown endpoint is intended for AI citation and retrieval pipelines.
Markdown packs use stable sections: At a Glance, Facts, Stops Table, Image
References, Citation URLs, and Do-Not-Infer Notes. The AI visibility endpoint
returns suggested queries, answer cards, missing metadata, weak citations,
image-alt gaps, and recommended copy/API improvements.

For Chinese retrieval, call the same endpoint with `?lang=zh`; markdown section
headings and user-facing text localize, while slugs, handles, JSON keys, and
error codes stay stable.

## Authenticated Endpoints

- `POST /api/v1/ai/describe-photo` requires `ai:run`
- `POST /api/v1/ai/compose-project` requires `ai:run`
- `POST /api/v1/ai/generate-postcard` requires `ai:run`
- `POST /api/v1/projects/sync` requires `project:write`

AI text routes accept `locale` or `outputLocale` in the JSON body. Image model
prompts may remain internally English; returned labels/copy follow the requested
output locale.

## MCP

The `/mcp` endpoint exposes the same capability set through JSON-RPC style MCP
methods. Public read tools require no auth. AI and write tools require auth and
validate the request `Origin` header when one is present.

Public MCP tools include `audit_public_project_visibility`, which wraps
`GET /api/v1/projects/{handle}/{slug}/ai-visibility`.

MCP accepts locale through query/header/cookie or tool/prompt arguments
(`locale` / `outputLocale`). Method names and tool names are not translated;
descriptions and returned public content are.

## OpenAPI

`GET /api/openapi.json` is generated by `web/lib/openapi.ts`. Use
`/api/openapi.json?lang=zh` for localized summary/description text. Keep it in
sync with API route behavior when changing v1 endpoints.
